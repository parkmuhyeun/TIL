컴퓨팅 자원 

- Amazon AWS
a. AWS는 가장 광범위하고 심층적인 컴퓨팅 기능을 제공합니다. Amazon Elastic Cloud Compute(EC2)는 다양한 프로세서, 스토리지 및 네트워킹을 선택하여 인프라를 관리할 수 있는 세분화된 제어 기능을 제공합니다.

b. AWS는 최신 기술을 기반으로 빠르게 구축, 배포 및 출시할 수 있는 여러 방법을 제공합니다.

c. AWS는 AWS 다음의 대형 클라우드 공급자보다 훨씬 더 많은 보안, 규정 준수 및 거버넌스 서비스와 주요 기능을 제공합니다.

d. AWS 컴퓨팅을 사용하면 장기 계약이나 복잡한 라이선스 없이 필요한 인스턴스 또는 리소스에 대해 사용한 기간에 해당하는 비용만 지불하면 됩니다. AWS는 가격 대비 성능을 개선하는 데 도움이 되는 자동 권장 사항과 비용을 추가로 최적화할 수 있는 도구 및 혁신적인 요금 모델을 제공합니다.

- MS Azure
애플리케이션을 새로 구축할 때나 기존 애플리케이션을 배포할 때, Azure는 앱을 실행하는 데 필요한 인프라를 제공합니다. 클라우드의 용량을 컴퓨팅하고 필요시 확장할 수 있습니다. 

애플리케이션을 컨테이너화하고, Windows 및 Linux VM(가상 머신)을 배포하고, VM을 Azure로 마이그레이션하기 위한 유연한 옵션을 활용합니다. 하이브리드 환경에 대한 포괄적인 지원을 통해 원하는 방법으로 원하는 위치에 배포합니다.

 Azure 컴퓨팅에는 모든 기능이 갖춰진 솔루션이 포함되므로 관리되는 엔드포인트 보호와 함께 온-프레미스 및 클라우드 앱에 안전하게 액세스할 수 있는 Active Directory 지원을 받습니다. 종량제 가격 및 Azure 하이브리드 혜택을 통해 앱을 배포하고 비용을 절감할 수 있습니다.

- Google GCP
a. Compute Engine을 사용하면 맞춤설정 가능한 가상 머신을 만들고 실행할 수 있습니다. Tau VM은 수평 확장 워크로드에 동급의 다른 클라우드 제품보다 42% 더 우수한 가격 대비 성능을 제공합니다.

b. Google Kubernetes Engine 또는 Cloud Run을 사용해 자동으로 컨테이너를 배포, 확장, 관리할 수 있습니다.

c. 가상 머신으로 직접 이전하거나 코드 재작성 없이 자동으로 컨테이너로 현대화하여 원하는 속도로 앱을 마이그레이션할 수 있습니다.

- IBM Cloud 
a. 고성능 서버
워크로드 요구사항에 맞게 구성할 수 있는 강력한 클라우드 서버를 통해 즉시 온라인 상태가 됩니다.

b. 최신 글로벌 데이터 센터
6개 지역의 60개 클라우드 데이터 센터와 PoP에서 빠르게 구동하여 실행합니다.

c. Kubernetes 관리
PaaS(Platform-as-a-Service)를 사용하여 개별 마이크로서비스를 클라우드 환경에 배치합니다.

d. 타협 없는 서버리스
거의 무제한으로 코드를 실행하고 사용량 만큼만 비용을 지불합니다. 심지어는 비용을 제로(0)로 낮춥니다.

네트워킹 

- Amazon AWS
a. 가장 뛰어난 보안
AWS 네트워킹 기능은 전 세계에서 가장 엄격한 보안 요구 사항을 충족하도록 설계되었습니다. 24/7 인프라 모니터링을 통해 데이터의 기밀성, 무결성 및 가용성을 보장합니다.

b. 최고의 네트워크 가용성
AWS 리전/가용 영역 (AZ) 모델을 사용하여 미션 크리티컬 워크로드의 가용성을 최고 수준으로 유지할 수 있습니다. 업계 전문 분석 기관에서는 고가용성이 필요한 엔터프라이즈 애플리케이션을 실행할 때 AZ 모델을 사용할 것을 권장합니다.

c. 일관된 고성능
최고 수준의 처리량과 최저 수준의 대기 시간을 제공하는 클라우드 네트워크를 사용하여 워크로드를 실행할 수 있습니다. AWS 네트워크를 사용하면 더 빠르고 응답성이 높은 애플리케이션을 고객에게 제공할 수 있습니다.

d. 가장 광범위한 글로벌 서비스 지원
목적별 네트워크를 통해 전 세계 모든 위치에서 애플리케이션과 콘텐츠를 제공하고 연결할 수 있습니다. AWS는 어떤 공급자도 따라올 수 없는 최대 규모의 글로벌 인프라 공간을 제공합니다.

- MS Azure
a. 연결 서비스
Azure의 VNet(Virtual Network), Virtual WAN, ExpressRoute, VPN Gateway, Virtual Network NAT Gateway, Azure DNS, Peering Service 및 Azure Bastion에서 네트워킹 서비스 중 일부 또는 조합을 사용하여 Azure 리소스 및 온-프레미스 리소스를 연결합니다.

b. 애플리케이션 보호 서비스
Azure에서 Load Balancer, 프라이빗 링크, DDoS 보호, 방화벽, 네트워크 보안 그룹, 웹 애플리케이션 방화벽 및 가상 네트워크 엔드포인트와 같은 네트워킹 서비스의 조합을 사용하여 애플리케이션을 보호합니다.

c. 전달 서비스
Azure의 CDN(Content Delivery Network), Azure Front Door Service, Traffic Manager, Application Gateway, Internet Analyzer 및 Load Balancer에서 네트워킹 서비스의 조합을 사용하여 Azure 네트워크에서 애플리케이션을 전달합니다.

d. 네트워크 모니터링
Azure Network Watcher, ExpressRoute Monitor, Azure Monitor 또는 VNet TAP(터미널 액세스 지점)에서 네트워킹 서비스 중 일부 또는 조합을 사용하여 네트워크 리소스를 모니터링합니다.

- Google GCP
a. Cloud Armor를 사용하면 애플리케이션과 웹사이트를 서비스 거부 및 웹 공격으로부터 보호할 수 있습니다.

b. Cloud CDN을 사용하면 웹 및 동영상 콘텐츠 전송을 빠르고 안전하게 확장할 수 있습니다.

c. VPN, 피어링, 엔터프라이즈 지원 등 하이브리드 연결 옵션을 지원합니다.

- IBM Cloud 
IBM은 다음을 포함한 추가 네트워킹 솔루션을 제공합니다.

a. 게이트웨이 어플라이언스는 네트워크 트래픽에 대한 제어를 강화하고, 네트워크 성능을 향상시키며, 네트워크에 보안 기능을 강화할 수 있도록 해주는 디바이스입니다. 다중 VLAN 라우팅, 방화벽, VPN, 트래픽 셰이핑 등을 위한 실제 및 가상 네트워크를 관리합니다.

b, 직접 링크는 개인용 인프라, 멀티클라우드 및 IBM Cloud 간의 데이터 전송을 보호하고 가속화합니다.

c. 클라우드 인터넷 서비스는 클라우드에 도달하기 전에 공공 대면 웹 콘텐츠와 애플리케이션을 보호하도록 설계된 보안 및 성능 기능입니다. 클라우드에 도달하기 전에 공공 대면 웹 콘텐츠와 애플리케이션을 보호하도록 설계된 보안, 안정성 및 성능 기능 세트와 DDoS 보호, 글로벌 로드 밸런싱을 활용하세요. 

스토리지 서비스 

- Amazon AWS
a. 필요한 스토리지에 더 빠르게 액세스
몇 주가 아닌 몇 분 안에 리소스를 제공할 수 있습니다. 클릭 몇 번으로 출시를 앞당기고 복잡한 용량 계획을 방지하며 오버프로비저닝을 줄일 수 있습니다.

b. 스토리지 지출 절감
인프라 유지 관리가 필요하지 않은 관리형 서비스를 사용하여 총 소유 비용(TCO)을 최소화할 수 있습니다. 데이터 액세스 빈도 및 속도에 따라 스토리지 비용을 최적화할 수 있습니다.

c. 스토리지 보안
AWS는 다른 클라우드 공급자보다 많은 보안 표준 및 규정 준수 인증을 지원하여 전 세계 거의 모든 규제 기관의 요구 사항을 충족할 수 있도록 돕습니다.

d. 혁신을 지원하는 새로운 인사이트
다양한 도구 중에서 선택하여 데이터 활용도를 높이고 새로운 제품 및 서비스를 더 빠르게 제공할 수 있습니다. 모든 클라우드 데이터에서 빅 데이터 분석, 인공 지능(AI), 기계 학습(ML), 고성능 컴퓨팅(HPC) 및 미디어 처리 애플리케이션을 실행할 수 있습니다.

- MS Azure
Azure Storage는 클라우드의 다양한 데이터 개체에 대해 고가용성, 대규모 확장성, 내구성 및 보안 스토리지를 제공합니다. Azure Storage 데이터 개체는 REST API를 통해 HTTP 또는 HTTPS를 통해 전 세계 어디에서나 액세스할 수 있습니다.

 Azure Storage는 .NET, Java, Python, JavaScript, C++ 및 Go를 사용하여 애플리케이션 또는 서비스를 빌드하는 개발자를 위한 클라이언트 라이브러리도 제공합니다. 개발자와 IT 전문가는 Azure PowerShell 및 Azure CLI를 사용하여 데이터 관리 또는 구성 작업을 위한 스크립트를 작성할 수 있습니다. Azure Portal 및 Azure Storage Explorer는 Azure Storage와 상호 작용하기 위한 사용자 인터페이스 도구를 제공합니다.

이점
- 내구성 및 고가용성
- 보안
- 확장 가능
- 관리됨
- 액세스 가능

- Google GCP
a. 객체 스토리지로 양에 상관없이 모든 유형의 데이터를 저장하고 원할 때마다 데이터를 가져올 수 있습니다.

b. Storage Transfer Service와 Transfer Appliance를 포함한 온라인 및 오프라인 전송 솔루션으로 데이터를 전송할 수 있습니다.

c. Persistent Disk 블록 스토리지는 Compute Engine 및 GKE 같은 Google Cloud 제품과 완전히 통합됩니다.

- IBM Cloud 
IBM의 클라우드 스토리지 서비스는 기존 및 클라우드 네이티브 워크로드를 지원하면서도 사용자 자신의 데이터에 대한 확장형의 보안이 강력하고 비용 효율적인 홈을 제공합니다. 오브젝트, 블록, 파일 스토리지 액세스와 같은 서비스를 프로비저닝 및 배포합니다. 요구사항 변경에 따라 용량을 조정하고 성능을 최적화합니다. 필요한 클라우드 스토리지에 대해서만 비용을 지불합니다.

a. 비용 효율성
당사의 유연한 티어 옵션과 무료로 제공되는 사설 네트워크를 이용하여 클라우드 스토리지 서비스에 대한 지출을 최적화합니다. IBM 데이터 센터 간에 데이터를 이동합니다.

b. 보안
BYOK 또는 IBM Cloud® Hyper Protect Services 키 관리를 통해 데이터를 보호합니다. IBM Cloud는 FIPS 140-2 레벨 4 하드웨어를 지원합니다.

c. 거버넌스
모든 클라우드 스토리지 환경에서 개선된 거버넌스와 식별 기능을 통해 정책을 관리하고 시행합니다.

d. 내구성
IBM Cloud Object Storage에 대해 99.99999999%의 데이터 내구성을 달성하고 내장형 내결함성 기능으로 데이터 복원력을 높입니다.
