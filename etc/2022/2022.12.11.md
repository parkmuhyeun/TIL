```
웹 이전 시대 (1985-1994)
- 인터넷은 소수의 전문가만 사용

제 1차 웹 시대 (1995-2000)
- 인터넷의 대중화 및 웹 기반 비즈니스 사업 모형 등장
- 서버급 하드웨어를 사용하여 자사 웹 싸이트 제공
- 가용성 및 유지 보수가 중요한 회사 성패의 요소 (24*7)
- 부하 분산기(load balancer)가 등장한 시기

닷컴 붕괴 시대 (2000-2003)
-높은 초기 하드웨어 및 소프트웨어 투자 비용으로 회사 기반 붕괴
- 하드웨어가 일용품화 됨, 세계적 네트워크 백본이 완성됨
- 높은 가용성의 요구와 분산 시스템의 대두 (분산 컴퓨팅)

제 2차 웹 시대 (2003-2010)
- 분산 컴퓨팅의 대중화
- 새로운 웹 활용 문화 탄생

클라우드 컴퓨팅 시대 (2010 - 현재)
- 분산 시스템 활용을 통한 투자 비용의 개념 전환
- 소규모 컴퓨팅에서의 활용

클라우드 컴퓨팅의 역사
- Amazon에서 시작
- 서버가 별다른 트래픽 없이 유휴상태인 경우가 다수 발생
- 고객에게 임대해 주고 사용한 시간만큼만 이용료를 받자는 개념
- AWS (Amazon Web Service)

클라우드 도입의 필요성
- 변화의 속도 : 게임 생명 주기는 약 2주 ~ 6개월 정도에 불과
- 몇 주 사이에 시스템을 구축하고 또 폐기 하여야 한다.

컴퓨팅 모델의 진화
메인프레임 기반 컴퓨팅 -> 클라이언트/서버 컴퓨팅 -> 클라우드 컴퓨팅

Utility Computing
전기나 네트워크 같은 시설로서의 기반 기설 개념의 클라우드 컴퓨팅

클라우드 컴퓨팅의 정의
원격으로 확장성 있고 측정된 IT 자원을 제공하기 위한 목적으로 고안된 별개의 IT 환경

클라우드 이전의 클라우드 토대를 쌓은 개념
클러스터 컴퓨팅 (Cluster Computing)
- 여러 대의 컴퓨터들이 연결되어 하나의 시스템처럼 동작하는 컴퓨터들의 집합

그리드 컴퓨팅(Grid Computing)
- 컴퓨팅 자원이나 하나 이상의 논리적인 풀로 구성된 플랫폼

클라우드 컴퓨팅의 특징
온디맨드(On-Denamd)
- 과거의 서비스는 제공자 중심
- 온디맨드는 철저히 소비자 중심의 개념
-  필요할 때, 필요한 만큼의 서비스를 요청

탄력성 (elasticity)
- Scale-Out
- 확장성, 가용성, 신뢰성
- Auto-Scaling
- 신속한 확장과 축소

민첩성 및 신속한 구현
- 주문 제작  IaaS, PaaS, SaaS

셀프 서비스
- 직접 생산하는 것이 아니라, 요청과 사용의 개념에서의 셀프 서비스

사용량 기반 과금을 통한 측정된 서비스(measured service)
- 할당된 자원, 사용 시간, 보안 수준 등을 기준으로 과금

Scale-Out vs Scale-Up
Scale Out: 자원 확장
Scale Up: 성능 확장

이용률
- 대부분의 기업 하드웨어 장비의 이용률은 30% 이하
-  95% 이상은 노후 장비의 재활용도 불가능
- 시간대 별로도 이용률이 달라질 수 있음
-> 자원 풀링을 통한 다중 임대(multi-tenant) 방식으로 서비스 제공

멀티 테넌시(tenancy) : 다중 자원 임대
- 여러 소비자에게 하나의 프로그램 인스턴스를 각각 제공하여, 각 소비자가 독립적
으로 이를 사용할 수 있도록 해 주는 기술

투자 및 비용 절감

보안 강화
- 신뢰성 있는 클라우드 서비스 제공자의 강화된 보안 수준 확보 및 활용 가능
 
이동성
- 언제 어디서나 접속 가능

클라우드 컴퓨팅의 위험 및 도전 과제
- 보안 취약성 증가
- 운영 관리에 대한 제어권 감소
- 클라우드 소비자가 클라우드 서비스 업체 변경을 원할 시,
  - 클라우드 제공자들간의 이식성 제한
  - 표준안 부족으로 클라우드간에는 일반적으로 배타적
- 다중 영역 준수 및 법적 이슈

클라우드 서비스 모델
IasS - 서버, 스토리지, 네트워크, 보안 같은 인프라를 네트워크를 통해 서비스를 제공하는 형태
PaaS - 응용 소프트웨어를 개발하기 위한 언어, 도구, 형상 관리, 배포 관리, 플랫폼, 미들웨어를 네트워크를 통해 서비스를 제공하는 형태를 말한다.
SaaS  - 서비스 제공자가 네트워크를 통해 사용자가 요구하는 형태의 응용 소프트웨어를 제공하는 형태

SaaS(software as a service)
- 웹 사이트 형태로 제공되는 응용 프로그램

SaaS를 뒷받침한 기존 개념 1
Mashups
- 웹서비스 업체들이 제공하는 각종 콘텐츠와 서비스를 융합하여 새로운 웹서비스를 생성 하는 것

SOA (service oriented architecture)
- 소프트웨어 인프라를 구축하는 방법론 가운데 하나
- 정보 시스템을 서비스 단위로 구축하여 다른 서비스와 연동 및 통합

SaaS 사례
- Salesforce.com, Office 365

PaaS(platform as a service
- 외부 공급업체가 제공한 소프트웨어 개발 플랫폼
- 이미 배포되고 설정된 IT 자원으로 구성된, 사용할 준비가 되어 있는 환경

PaaS 사례
- Google App Engine, Microsoft Azure

IaaS(Infrastructure-as-a-service)
- 사용할 준비가 된 컴퓨터 및 네트워크 하드웨어
- 사용자가 필요로 하는 서버, 계산능력, 스토리지, 네트워크 기능 제공

Iaas 사례
- Amazon Ec2, S3, DynamoDB

클라우드 서비스 모델 비교
- 클라우드 소비자의 제어 수준
SaaS: 사용 및 사용 관련 설정
PaaS: 제한적인 관리
IaaS: 모든 관리
- 클라우드 소비자가 사용할 수 있는 기능
SaaS: 프론트 엔드의 사용자 인터페이스에 접근
PaaS: 플랫폼 사용과 연관된 IT 자원에 대한 중간 수준의 관리자 권한 제어
IaaS: 가상 인프라와 관련된 IT 자원에 대한 완전한 접근
```
