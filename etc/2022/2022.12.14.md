```

클라우드 컴퓨팅 배치 모델
퍼블릭 클라우드(Public Cloud)
- 판매 기관이 소유하고 있는 클라우드
- 클라우드 제공자가 소유하고 공개적으로 접근 가능한 클라우드

프라이빗 클라우드(Pricate Cloud)
- 특정 기관이 직접 구축하고, 해당 기관 자체를 위해 운영되는 클라우드
- 하나의 조직이 자신의 클라우드를 개별적으로 소유

하이브리드 클라우드
- 민감한 데이터와 덜 민감한 데이터를 구분

커뮤니티 클라우드
- 특정 커뮤니티 클라우드 소비자에게만 접근이 허용

가상 프라이빗 클라우드 (Virtual Private Cloud)
- 퍼블릭 클라우드가 제공해주는 프라이빗 클라우드 형태

개인용 클라우드 (Personal cloud)
- 개인 장치에 장착된 앱에서 제공하는 클라우드 서비스로, 자기 자신의 개인적 내용이 저장되는 공간
- ex) youtube, Facebook, Twitter

모바일 클라우드
- 단말기 상에서 클라우드 서비스를 이용

멀티 클라우드
- 2개 이상의 클라우드를 상호 연결하는 기술

가상 머신 (VM)
- 하나의 시스템에 생성된 독자적 시스템 인스턴스

하이퍼바이저(hypervisor)
- 하나의 물리 서버에 여러 개의 가상 머신을 생성하고 관리하는 프로그램

가상화
- 하나의 물리 서버에 하나 이상의 가상 머신을 생성하여 복수개의 논리서버를 운영하는 기술

프로비저닝(provisioning)
- 사용자가 원하는 형태의 가상 머신을 만들어 사용자에게 제공할 준비를 수행하는 과정

스케일 업의 문제점
- 시스템 크기 증가에 한계
- 경제성
- 스케일 업이 통하지 않는 경우 발생
  - ex) 단일 쓰레드로 실행하는 소프트웨어 

클라우드 서비스
- 클라우드를 통해 원격 접근이 가능한 임의의 IT 자원

클라우드 서비스 소비자
- 소프트웨어 프로그램이에 의해 부여되는 일시적인 런타임 역활
- 소프트웨어 프로그램이나 워크스테이션 혹은 모바일 장치

클라우드 서비스 소유자
- 법적으로 클라우드 서비스를소유하는 개인이나 조직
- 클라우드 소비자나 제공자 모두 클라우드 서비스 소유자일 수 있다.

클라우드 자원 관리자
- 클라우드 소비자 조직 혹은 제공자 조직 모두 자원 관리자가 될 수 있다.

클라우드 감사
- 클라우드 확녕에 대해 독립적 평가를 수행하는 제 3자 인증 기관

클라우드 운반자
- 네트워크와 전기통신 제공자

영내 (On-Premise) : 온프레미스
- 기업 조직의 경계 내부
- 클라우드 기반이 아닌 기업 내부의 통제된 IT 환경의 영역을 주로 지칭

조직 경계와 신뢰경계
- 조직 경계: IT 자원을 둘러싸고 있는 물리적 경계
- 신뢰 경계: IT 자원을 신뢰할 수 있는 범위에 대한 논리적 경계

클라우드 플랫폼
- 사용자 요청에 따라 클라우드 서비스를 제공하는 관리 체계
컨트롤러 노드
- 클라우드 플랫폼이 설치되어 클라우드 서비스 인프라를 관리하는 기능을 제공하는 서버
컴퓨트 노드
- 연산 능력을 제공하는 장비로, 하이퍼바이저가 설치되고 가산 머신이 생성되는 서버
스토리지 노드
- 가상 머신에 저장소를 제공하기 위한 서비스 서버
네트워크 노드
- SDN 서비스나 가상 라우터 서비스를 제공하는 서버

셀프서비스 포털
- 클라우드 서비스를 이용할 수 있는 웹 서비스로, 클라우드 자원에 대한 카탈로그를 제공하고, 서비스 신청과 과금 기능 제공
운영자 포털
- 클라우드 서비스 인프라에 대한 모니터링, 장애, 용량, 과금, 관ㄹ 등의 관리와 운영을 위한 서비스
미터링과 과금
- 클라우드 서비스의 자원 사용량을 계측하는 행위가 미터링, 사용량 기반의 비용을 청구하는 것이 과금
클라우드 마이그레이션
- 물리 장비에서 클라우드 서비스로 이관하거나 가상화에서 클라우드 서비스로 운영중인 시스템을 이관하는 서비스

블록 스토리지
- 인스턴스 단위로 추가 할당 삭제가 이루어지는 경우 사용되는 스토리지

오브젝트 스토리지
- 사용자 계정별로 저장 공간 할당
- ex) 네이버 N 드라이브, Dropbox

가상화
- 물리적 자원과 사용자가 사용하는 자원의 분리
- 장점
- 서버 이용률 향상
- 각 사용자의 수행 환경을 다른 사용자로부터 고립
- 통합 가능
- 이동성 증가
- 하드웨어 모의실험 기능 제공

클라우드 환경에서 가상화가 적합한 이유
- 가상 머신의 동적 관리
- 하드웨어 독립성
- 서버 통합
- 경제성

가상화의 분류(가상화 구현 층에 따른 분류)

하드웨어 기반 가상화
- 물리자원과 운영체제 사이에 VMM이 존재
- type1

운영체제 기반 가상화
- OS 내부에서 가상 자원을 생성하고 guest OS가 동작
- Host OS -> VMM -> Guest OS
- OS가 물리 자원에 대한 추상화 기능 제공
- type2

미들웨어 기반 가상화
- OS 윗 단의 런타임 라이브러리 수준에서 가상화 제공
- JAVA VM
  - 추상화와 가상화 제어의 주체가 미들웨어
```
