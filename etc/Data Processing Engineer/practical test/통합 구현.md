```
3장 통합구현

통합구현
송 수신 모듈과 중계 모듈 간의 연계를 구현하는 것

통합구현 구성 요소
송신 시스템과 모듈
수신 시스템과 모듈
중계 시스템
연계 데이터
네트워크

연계 요구사향 분석
연계 데이터를 식별 및 표준화하여 연계 데이터를 정의하는 것

연계 요구사항 분석 절차
1. 하드웨어 및 소프트웨어 구성, 네트워크 현황 확인
2. 테이블 정의서, 코드 정의서 등 문서 확인
3. 체크리스트 작성
4 관련 문서 공유 및 인터뷰 설문 조사 실시
5. 요구사항 정의서 작성

연계 데이터 식별 및 표준화의 절차
1. 연계 범위 및 항목 정의
2. 연계 코드 변환 및 매핑
3. 연계 데이터 식별자와 변경 구분 추가
4. 연계 데이터 표현 방법 정의
5. 연계 정의서 및 명세서 작성

연계 매커니즘의 연계 방식
직접 연계 방식: 중간 매개체 없이 송수신 직접, ex) DB Link, API/Open API, JDBC
간접 연계 방식: 중간 매개체 두어 연계, ex) 연계 솔루션, ESB, 소켓

연계 매커니즘의 과정
송신 시스템 및 송신 모듈로 부터
-> 연계 데이터 생성 및 추출
-> 코드 매핑 및 데이터 변환
-> 인터페이스 테이블 또는 파일 생성
-> 연계 서버 또는 송신 어댑터
전송
-> 연계 서버 또는 수신 어댑터
->인터페이스 테이블 또는 어댑터
-> 코드 매핑 및 데이터 변환
-> 운영 db에 연계 데이터 반영
수신 시스템 및 수신 모듈로

구간별 주요 오류 발생 시점
송신 시스템
- 데이터 생성 및 추출시
- 코드 매핑 및 데이터 변환시
- 인터페이스 테이블 또는 파일 등록시
수신 시스템
- 연계 데이터 로드시
- 코드 매핑 및 데이터 변환시
- 운영 db에 반영시
연계 서버
연계 데이터 로드 및 전송 형식으로 변환 시
연계 데이터 송 수신 시
수신 시스템 데이터 형식으로 변경 및 로드 시

장애 및 오류 유형의 분류
- 송 수신 시스템 연계 프록르ㅐㅁ 오류
- 연계 서버 오류
- 연계 데이터 오류
- 기록하는 로그를 통해 장애 및 오류 확인후 분석하여 적절한 처리방법 선택

장애 확인과 처리 절차
1차적으로 모니터링화면
1차에서 확인이 불가능한 경우 로그 테이블
오류 발생 원인 확인되면 적절한 조치

장애 및 오류의 정의와 설계
장애 및 오류 관리 대상
관리 대상의 장애 및 오류 코드와 메시지
장애 및 오류 코드와 메시지 관리 방식
장애 및 오류 기록 방식

연계 데이터 보안 방식
전송 구간 보안: 암호화 기능이 포함된 프로토콜 사용
데이터 보안: 추출할때나 반영할때 데이터 암복호화

데이터의 암 복호화 처리 절차
- 송신 시스템
1. DB에서 연계 데이터 추출
2. 보안 적용 대상 컬럼을 암호화
3. 연계 데이터를 인터페이스 테이블 또는 파일에 등록 및 수신
- 수신 시스템
4. 수신된 데이터 중 암호화된 컬럼을 복호화
5. 운영 DB에 반영

암 복호화 적용 절차
암호화 적용 대상, 암호화 알고리즘, 암호화 키 선정
-> 컬럼의 데이터 길이 변경
-> 알고리즘 라이브러리 확보 및 설치
-> 연계 응용 프로그램에서 암복호화 처리

XML
특수한 목적을 갖는 마크업 언어를 만드는 사용되는 다목적 마크업 언어

SOAP
네트워크 상에서 HTTP/HTTPS/SMTP 등을 이용하여 XML을 교환하기 위한 통신 규악

WSDL
웹 서비스와 관련된 서식이나 프로토콜을 등을 표준적인 방법으로 기술하고 게시하기 위한 언어

연계 테스트
연계 시스템과 연계 시스템의 구성 요소가 정상적으로 작동하는지 확인하를 활동

연계 테스트 진행 순서
-> 연계 테스트 케이스 작성
-> 연계 테스트 환경 구축
-> 연계 테스트 수행
-> 연계 테스트 수행 결과 검증
```
