최적 적합 프로그램이나 데이터가 들어갈 빈 영역 중 단편화를 가장 적게 남기는 곳에 배치
최악 적합: 프로그램이나 데이터가 들어갈 빈 영영 죽 단편화가 가장 많이 남기는 곳에 배치

교체 전략

주기억장치 할당
프로그램이나 데이터를 실행시키기 위해 주기억장치에 어떻게 할당할 것인가

주기억장치 할당기법 분류
연속 할당 기법: 프로그램을 주기억장치에 연속하여 할당하는 기법(단일 분할 할당기법, 다중 분할 할당 기법)
분산 할당 기법: 프로그램을 특정 크기로 나누어 분산하여 저장하는 할당하는 기법(종류: 페이징 기법, 세그먼테이션 기법)

단일 분할 할당 기법
한순간에는 오직 한명의 사용자만이 주기억 장치의 사용자 영역을 사용
단일 분할 할당 기법 종류: 오버레이: 주기억 장치 보다 큰 프로그램을 실행하기 위한 기법, 스와핑: 하나의 프로그램 전체를 주기억 장치에할당하여 사용하다 필요에 따라 다른 프로그램과 교체하는 기법

경계 레지스터: 사용자 영역에 있는 사용자가 운영체제 영역에 접근하지 못하도록 하는 기법

다중 분할 할당 기법
종류
고정 분할 할당 기법(정적 할당 기법): 미리 고정된 크기 분할하여 할당
가변 분할 할당 기법(동적 할당 기법) : 미리 분할하지 않고 필요에 따라 분할

가상기억장치: 보조기억장치의 일부를 주기억장치처럼 사용하는 것

페이징 기법
프로그램과 주기억장치의 영역을 동일하게 나눈 후 나눠진 프로그램을 주기억 장치에 적재하여 실행하는 기법
외부 단편화는 발생하지 않지만, 내부 단편화 발생

세그멘테이션 기법
프로그램을 다양한 크기의 논리 단위로 나눈 후 주 기억장치에 적재시켜 실행하는 기법
내부 단편화는 바생하지 않지만 외부단편화는 발생

페이지 교체 알고리즘
페이지 부재 발생하면 가상기억장치에서 필요한 페이지를 적재해야되는데, 이 때 주기억장치의 모든 페이지가 사용중이라면 어떤 페이지 프레임을 선택하여 교체할지 결정하는 기법

페이지 부재: CPU가 액세스한 페이지가 주기억장치에 없는 경우

최적교체(OPTimal replacement)
앞으로 가장 오랫동안 사용하지 않을 페이지를 교체

FIFO(First in First Out)
가장 먼저 들어와서 오래 있떤 페이지를 교체하는 기법

LRU(Least Recently Used)
최근 가장 오랫동안 사용하지 않은 페이지를 교체하는 기법

LFU (Least Frequently Used)
사용빈도가 가장 적은 페이지 교체하는 기법

NUR(Not Used Recently)
최근에 사용하지 않은 페이지를 교체

SCR(Second Chance Replacement) 2차 기회 교체
가장 오랫동안 주기억장치에 있던 페이지 중 자주 사용하는 페이지의 교체를 방지하기 위한 기법, FIFO 단점 보완

페이지 크기
페이지 크기가 작을 경우
페이지 단편화 감소, 한개의 페이지를 주기억장치로 이동시키는 시간이 줄어듦, 디스크 접근횟수가 많아져서 전체적인 입/출력 시간은 늘어남

페이지 크기가 클 경우
페이지 단편화 증가, 한개의 페이지 주기억장치로 이동시키는 시간이 많이듬, 디스크 접근횟수가 줄어들어 전체적인 입/출력 효율성 증가

Locality
프로세스가 실행되는동안 주기억장치를 참조할때 일부 페이지만 집중적으로 참조하는 성질

시간 구역성(Temporal Locality) - 프로세스가 실행되면서 하나의 페이지를 일정시간동안 집중적으로 액세스하는현상
공간 구역성(Spartial Locality) - 프로세스가 실행시 특정 위치의 페이지를 집중적으로 액세스하는 현상

워킹 셋
프로세스가 일정시간 동안 자주 참조되는 페이지들의 집합

스레싱

스케줄링 종류
장기 스케줄링
어떤 프로세스가 시스템의 자원을 차지할 수 있을 것인가 결정하여 준비 상태 큐로 보내는 작업

종류: FCFS,  SJF, 우선순위, HRN, 기한부

우선순위가 높은 다른 프로세스가 CPU를 강제로 빼앗아 사용할 수 있는 스케줄링 기법
종류: ROUND ROBIN, SRT, 선점 우선 순위, 다단계 큐, 다단계 피드백 큐

FCFS
준비 상태 큐에 도착한 순서에 따라 CPU를 할당하는 기법

대기시간: 프로세스가 대기한 시간
반환 시간: 프로세스 대기 시간 + 실행 시간의 합

SJF(Shortest Job First) 단기 작업 우선
실행 시간이 가장 짧은 프로세스 먼저 CPU를 할당하는 기법

HRN(Highest Response-ratio Next)
대기시간과 실행 시간을 이용하는 기법, 우선순위 계산하여 높은것 부터  ㅓ리
우선순위 계산식 = 대기시간 + 실행시간 / 서비스 시간
