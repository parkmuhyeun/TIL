# HTTP METHOD

GET

GET은 **데이터를 조회하기 위해 사용하는 방식**으로 데이터를 **쿼리 파라미터**를 통해 전달합니다. **URL  에 데이터가 노출**되기 떄문에 보안적으로 중요한 데이터를 포함해서는 안됩니다.

POST

POST는 **데이터를 추가 또는 수정**하기 위해 사용하는 방식으로 데이터를 **바디에 추가**해서 전송하는 방식입니다. 완전히 안전한 건  아니지만 URL에 데이터가 노출되지 않아 **GET보다는 안전**합니다.

PUT

PUT은 **리소스를 대체**하는데 **리소스가 있다면 대체**하고 **없으면 생성**합니다. 근데 여기서 대체할 때 **부분대체 하는 것이 아니라 완전히 대체**하기 때문에 주의해서 사용해야합니다.

PATCH

PATCH는 **PUT과 달리 리소스의 부분만 변경**하게 됩니다.

DELETE

DELETE는 **리소스를 삭제할 때 사용하는 메소드**입니다.

안전

**호출해도 리소스를 변경하지 않는걸** 의미하고, **GET 방식**을 제외하곤 안전하지 않습니다.

멱등

**몇번을 호출하든 결과가 똑같다**는 의미로 POST와 PATCH를 제외한 **GET, PUT, DELETE**는 멱등입니다.

캐시가능

**응답결과로 캐시가 가능한 메서드**는 **GET, POST, PATCH**가 있습니다. POTS와 PATCH는 본문까지 고려해야 되서 실**제로는 GET정도**만 쓰입니다.

# HTTP Status Code

상태코드는 클라이언트가 보낸 요청의 **처리 상태**를 알려주는 기능을 합니다

1xx: 요청이 **수신되어 처리중**일 때 응답받는 코드입니다.

2xx: 요청이 **정상적으로 처리된경우** 응답받는 코드입니다.

3xx: 요청을 완료하기 위해서는 **유저의 추가적인 조치**가 필요해서 **리다이렉션**할 때 사용하게 됩니다.

4xx: **클라이언트쪽의 요청에 잘못된 문법** 등으로 발생하게 되는 응답 코드입니다.

대표적으로

- 400번 **Bad Request**는 **클라이언트에서 잘못된 요청**을 했을 때 발생하는 코드입니다.
- 401번 **UnAuthorized**는 클라이언트가 해당 **리소스에 대해 인증**이 필요한 경우 입니다.
- 403번은 **Forbidden**은 클라이언트가 **인증**은 되어있지만 **권한이 불충분**할 때 발생하는 코드입니다.
- 404번 **Not** **Found**는 요청한 **리소스가 서버에 없을때** 발생하는 코드입니다.

5xx: **서버쪽에서 오류**가 발생해서 응답받는 코드입니다.

# 공인 IP vs 사설 IP

**공인 IP**는 **전세계에서 유일한 IP**로 **인터넷 서비스 공급자가 제공하는 IP주**소입니다. **외부에 공개**가 되어있기 때문에 **인터넷에 연결된 다른 장비로부터 접근이 가능**합니다. 그에 따라 **방화벽 같은 보안 설정**을 해주어야 합니다.

사설 IP는 **어떤 네트워크안에서 사용되는 IP 주소**로 I**PV4의 부족**으로 인해 모든 네트워크가 공인 IP를 사용하는 것이 불가능하기 때문에 **네트워크 안에서 라우터를 통해 할당받는 가상의 주소**입니다. 별도의 설정 없이는 **외부에서 접근이 불가능**합니다.

# 쿠키와 세션

쿠키는 클라이언트 **로컬**에 저장되는 **키와 값으로 이루어진 파일로** **이름, 값, 유효 시간, 경로** 등을 포함하고 있습니다. 클라이언트 **상태 정보를 브라우저에 저장**하고 서버에 요청시 Cookie를 넘겨줄 수 있습니다. 쿠키는 **항상 서버로** 넘어가기 때문에 **최소한의 정보**만 사용하는게 좋고 서버에 전송하지 않고 웹 브라우저 내부에서 데이터를 저장하고 싶으면 **웹스토리지**를 사용하면 됩니다. 쿠키는 **사용자 로그인 세션** 관리나 **장바구니** 같은 곳에서 **활용**할 수 있습니다.

웹스토리지(로컬 스토리지, 세션 스토리지)

로컬스토리지: 브라우저를 닫아도 **남아있는** 스토리지

세션 스토리지: 브라우저를 닫으면 **사라지는** 스토리지

세션

일정시간 동안 **같은 브라우저로 들어오는 요청**을 **하나의 상태**로 보고 **그 상태를 유지하는 기술**입니다. 세션 같은 경우도 **쿠키를 사용해서 값을 주고받으며** 클라이언트 상태 정보를 유지하는데 쿠키에 **중요 정보는 넣지 않고 세션 아이디**를 넣어서 비교적 안전하게 저장하게 됩니다.

쿠키 vs 세션

쿠키 같은 경우 저장 위치는 **클라이언트**이고 세션은 **서버**에 저장되게 되기 때문에 **쿠키 같은 경우 보안에 취약**할 수 있습니다. 하지만 **세션 같은 경우 세션id만 쿠키에 저장하기 때문에 비교적 안전**합니다. 반대로 속도를 보면 쿠키는 **클라이언트에 저장되기 때문에 처리속도**가 빠르지만, 세션은 **실제 저장된 정보가 서버**에 있으므로 따로 **처리가 필요해 쿠키보다 느립니다**. 마지막으로 생명주기를 보면 **쿠키는 브라우저를 종료해도 계속해서 남아있을 수** 있지만 **세션 같은 경우는 만료시간에 상관없이 브라우저를 종료하면 삭제**되게 됩니다.
